OdbcDocument Class Reference

Inherits from NSPersistentDocument
Declared in OdbcDocument.h

Overview

The OdbcDocument class is inteded to use instead of XCode generated NSPersistentDocument class when creating a Core Data document-based project.

When creating a new application, uncheck ‘Create Document-Based application and uncheck also 'Use Core Data’ check box. Then let your document inherit from OdbcDocument like this:


// MyDocument.h

#import <Cocoa/Cocoa.h>
#import <Odbc/Odbc.h>

@interface MyDocument : OdbcDocument

@end

In your now empty MyDocument.m implement following methods:



// MyDocument.m

- (NSURL *) persistentStoreUrl {

    return [NSURL URLWithString : @"odbc:///testdb?username=root&password=secret"];
}

- (NSString *) windowNibName {

    return @"MyDocument"

This methods will be called by OdbcDocument to get ODBC database URL and name of document nib file.

The URL shown above is only an example. You should replace ‘testdb’ by your own ODBC data source name (DSN). You should replace ‘root’ with your username and you should replace ‘secrect’ by your own password.

The class name ‘MyDocument’ and nib name ‘MyDocument’ shown above are also only an example.

Methods to override

  persistentStoreUrl

This property returns ODBC URL. It should be overwriten by subclasses.

@property (readonly, nonatomic) NSURL *persistentStoreUrl

Return Value

ODBC URL for ODBC database to use.

Discussion

Returned URL should have following format:


odbc::///[data source name]/?username=[username]&password=[password]

where |data source name] is ODBC data source name, [username] is database username and [password] is users password.

Return Value

Core Data NSManagedObjectContext for the document.

Discussion

If you use NSArrayController for data access then in its Attribute Inspector in XCode Interface Builder you should set it to entity mode and specify the entity name. Next in Bindings Inspector in Managed Object Context you should bind to your using managedObjectContext as Model Key Path.
